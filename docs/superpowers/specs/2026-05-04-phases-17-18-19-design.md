## Phase 18: Production Readiness

### Requirements
| ID | Requirement | Notes |
|----|-------------|-------|
| PROD-1 | Structured logging with request IDs | Traceability |
| PROD-2 | Graceful shutdown on SIGINT/SIGTERM | No corrupted state |
| PROD-3 | Input validation hardening | Reject malicious payloads |
| PROD-4 | Health check includes all dependencies | LM Studio, GPU, disk space |
| PROD-5 | CORS restricted to known origins | Security |

### Design

**18.1 Structured Logging**
- Add `request_id` middleware
- Log all API calls with duration, status, request_id
- Write logs to `logs/udip.log` with rotation

**18.2 Graceful Shutdown**
- Handle SIGINT/SIGTERM in lifespan
- Cancel background tasks (TTL loop, broadcast loop, benchmark worker)
- Close all WebSocket connections
- Flush metrics history to disk

**18.3 Input Validation**
- Sanitize all string inputs (already done for kb_name/doc_id)
- Add max length limits (query ≤ 2000 chars, kb_name ≤ 64 chars)
- Reject binary payloads in JSON fields

**18.4 Enhanced Health Check**
- `GET /health`: Check LM Studio, GPU (pynvml), disk space (> 1GB free), VRAM monitor
- Return component statuses individually
- HTTP 503 if critical deps down

**18.5 CORS Hardening**
- Read `ALLOWED_ORIGINS` env var (default: localhost:3782, localhost:3000)
- Reject unknown origins in production
